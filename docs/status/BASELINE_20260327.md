# Waybroker Baseline Status Report (2026-03-27)

## 概要
Waybroker (TUFF-Xwin) の現状を調査し、P0-BASELINE 評価を完了した。
現在のコードベースは設計文書に基づいたプロセス分割（crates 分割）が完了しており、共通 IPC フォーマットと watchdog による監視・縮退遷移のコアロジックが正常に動作している。一方で、`compd` および `lockd` はバナー表示のみのスタブであり、表示ポリシーやロック制御の実体は未実装である。

## Runtime Artifact Inventory
`/run/user/1000/waybroker/` に生成された成果物：
- `active-profile.json`: 現在選択されているデスクトッププロファイル
- `launch-state-<profile>.json`: プロセス起動状態、PID、再起動回数、世代、シーケンス番号
- `watchdog-report-<profile>.json`: Watchdog による健康診断結果、推奨アクション（Restart/Degraded）
- `profile-transition-<from>-to-<to>.json`: 縮退遷移の記録（原因コンポーネントを含む）
- `*.sock`: 各サービスの Unix Domain Socket
- `*.log`: デモスクリプト実行時のログファイル

## 環境依存と既知の問題
- **スクリプトの実行権限**: リポジトリ直下のスクリプト群に実行権限が付与されていなかった（`chmod +x` で対応済み）。
- **PATH 依存**: `xfce-x11`, `openbox-x11` プロファイルはシステムに該当パッケージがインストールされていることを前提としている。未導入環境では `Missing` 状態として正しく判定される。
- **相対パス依存**: `demo-x11` プロファイル等はスクリプトを相対パス `./scripts/...` で参照しており、実行ディレクトリに依存する。
- **スタブ状態**:
    - `compd`: バナーを表示して終了するのみ。
    - `lockd`: バナーを表示して終了するのみ。
    - `displayd`: 常に固定の `eDP-1` 出力を返す stub。
- **ハードウェア未実装**: 本物の DRM/KMS、libinput、PAM との連携は未着手。

## 次のステップへの示唆
コアとなる `watchdog` / `sessiond` / `displayd` / `waylandd` の IPC 連携と状態遷移は既に堅牢に実装されており、P1 移行のタスク（`compd` / `lockd` の最小ランタイム化、プロファイル実行モデルの正規化）に進むための土台は整っている。
